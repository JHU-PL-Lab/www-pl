Our ML Dialect

There are more ML dialects out there than you would believe. We are going to use OCaml, version 4.02.1 (earlier versions will work as well for your own coding, but the binary distibutions of the FbDK are very picky on which version they are run under). OCaml is not completely compatible with the most standard dialect of ML, SML, but is nonetheless similar. It includes classes and objects, something not found in SML. We will often call OCaml just "Caml", you can view the two as synonyms.

Installing OCaml on Your Machine

OCaml is free and you should install it on your computer. Sources and binaries for OCaml for Windows, MacOS, Linux, etc are found on the OCaml downloads page.

OCaml is available on all department Linux machines via the shell command ocaml.

OCaml development environments

We strongly recommend you run OCaml from something besides the shell directly.
Eclipse OCaIDE
This is an Eclipse plugin for OCaml; it is the "official recommended method" and is what will be used in lecture. See information on using OCaIDE for more information.
Emacs
See information on running OCaml inside Emacs to get going.
vim OMLet
If you like vim, it now has a caml mode, OMLet. The above two modes are significantly better, all this mode offers is automatic indentation and keyword highlighting.
