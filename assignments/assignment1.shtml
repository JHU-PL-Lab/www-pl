You are to write several small programs in OCaml. Use the this template file which contains the questions and spots for your answers.

Resources to help you

Here is a reminder of some resources at your disposal.

  • Consult the code examples run in class.
  • Consult the Course OCaml page for information on installing OCaml and getting a good toolchain setup for development
  • We recommend you use the emacs caml mode to easily edit and debug OCaml code, but there are other options as well listed on the course OCaml page.
  • For more OCaml examples etc, read and copy/paste the code defined in the Caml manual Part I Chapter 1.
  • If you are looking for how some standard library function is expressed in OCaml, like not equal, etc, consult the Caml Pervasives which are the predefined functions available in OCaml.
  • You are strongly encouraged to work with other people on the assignment. You just need to list the names of people you worked with. However remember that you should submit your own write up of the answers. Copying or emailing of solutions is not allowed. For the full collaboration policy see here.
  • Come to the PL Collab Labs, in Malone 107, to get help from Prof, TAs, and your peers. Lab hours are on the contact us page.
  • There is also Piazza for online help and question clarification.

Submission and Grading

  • We will be using Gradescope to submit programming assignments. The Gradescope entry code is posted on Piazza.
  • For this and other programming assignments they will be autograded by Gradescope.
  • When asked to, upload your assignment1.ml file (that name exactly and no other files).
  • The tests we give in the HW as well as possibly some hidden tests will be run and your score will immediately be returned which is the sum of how you did on all the tests.
  • After all HWs are turned in we will show you the hidden tests and how you did on them, but not until then.
  • You can submit the HW as many times as you want up to the deadline. Any submissions after the deadline will fall under the late submission policy.
  • This is a new system and it is surely going to be buggy. Please submit your HW early and often to Gradescope, don't just wait until 10 minutes before the deadline, and report any problems/bugs to the instructors on Piazza.
  • Note that if one of your functions being tested has compile errors or is of the incorrect type for some of the test inputs, those tests won't appear at all in the output. The system will still continue running the remaining tests. If your program loops forever you may lose out on all the remaining points, so comment out any such infinite looper if there are questions after it that you want graded.
  • Please don't try to "game" the auto-grading system or hack it. Your grade will be 0 on the assignment if gaming is detected, and any hacking will be considered a serious ethics violation.
